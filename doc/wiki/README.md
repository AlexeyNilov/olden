# Wiki Reference Captures

This directory stores external wiki and reference captures used for planning.
These files preserve what the source said when captured or reviewed; they are
not canonical project requirements.

Promote only reviewed, committed behavior into the owner document:

- `doc/requirements.md` for testable system behavior.
- `doc/glossary.md` for shared project terms.
- `doc/roadmap.md` for future or deferred scope.
- `findings/` for exploratory analysis and local experiment notes.
